# Sprint Reviews

## Purpose

Sprint Reviews are historical records of completed sprints.

This hub indexes those records. It is not part of default onboarding.

## Audience

Maintainers and contributors reconstructing past sprint outcomes.

## Scope

- Post-sprint retrospective summaries
- Distinction from Architecture Reviews and Changelog

## Primary navigation

| Document | Sprint | Status |
|----------|--------|--------|
| `Sprint-6-4.md` | Recurring Rhythm Product Integration | Historical |
| `Sprint-6-5.md` | Primary Rhythm Ownership | Historical |
| `Sprint-8.md` | Today Product Experience | Historical |

All Sprint Reviews in this folder are archived.

## Related historical records

Completed work is also summarized in `Docs/CHANGELOG.md`.

## What this hub does NOT contain

- Current product contracts
- Active architecture or design authority
- Living roadmap status
- Day-to-day implementation guidance
- Pre-sprint architecture contracts

For current behavior, use `Docs/Product/` and `Docs/Design/`.  
For current progress, use `Docs/ROADMAP.md` and `Docs/CHANGELOG.md`.
For pre-sprint architecture history, see `Docs/Architecture/Reviews/README.md`.
