# Architecture Reviews

## Purpose

Architecture Reviews capture the pre-implementation architecture contract for a specific Sprint.

They are immutable historical records.

## Audience

Maintainers and architecture reviewers reconstructing past sprint constraints.

## Scope

- Architectural scope agreed before implementation
- Constraints and completion criteria for that Sprint

Each Sprint may have at most one Architecture Review.

## Primary navigation

| Document | Sprint | Status |
|----------|--------|--------|
| `Sprint6-5_ArchitectureReview.md` | Sprint 6-5 Primary Rhythm Ownership | Historical |

All Architecture Reviews listed here are archived and are not part of default onboarding.

## What this hub does NOT contain

- Accepted long-term Decision Records
- Current Product or Design contracts
- Post-sprint retrospective narrative

Decision Records: `Docs/Architecture/Decisions/`  
Current structure: `Docs/Architecture/ARCHITECTURE.md`
