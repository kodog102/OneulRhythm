# Development

## Purpose

Development documentation defines how OneulRhythm work is planned, implemented, validated, and closed.

## Audience

Developers and AI implementation agents.

## Scope

- Official Sprint workflow
- Cursor execution rules
- Prompt library
- QA pipeline
- Sprint close checklist
- Links to Engineering Charter and AGENTS

## Primary navigation

| Document | Role |
|----------|------|
| `DEVELOPMENT_WORKFLOW.md` | Official Sprint stages and roles |
| `CURSOR_GUIDELINES.md` | Project-wide Cursor execution rules |
| `PROMPT_LIBRARY.md` | Reusable implementation prompts |
| `QA_PIPELINE.md` | QA process |
| `SPRINT_CHECKLIST.md` | Sprint completion checklist |

Related authoritative documents outside this folder:

- `Docs/Engineering/ENGINEERING_CHARTER.md` — engineering principles
- `Docs/AI/AGENTS.md` — product and architecture rules for AI agents

1. `Docs/AI/AGENTS.md`

## What this hub does NOT contain

- Product philosophy or UX specifications
- Architecture structure or Decision Records
- Design implementation contracts
- Feature roadmaps beyond priority checks in `ROADMAP.md`
